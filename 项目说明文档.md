# 新闻与论坛互动平台 — 项目说明文档

## 一、项目概述

本项目是一个**新闻与论坛互动平台**，采用前后端分离架构，包含用户前台和管理员后台两套界面。平台提供新闻浏览与发布、论坛社区互动、问卷调查、公告资讯、在线留言等功能，并基于协同过滤推荐算法为用户提供个性化新闻推荐。

### 系统组成

| 模块 | 技术栈 | 端口 | 说明 |
|------|--------|------|------|
| backend | Spring Boot 3.3.3 + MyBatis-Plus | 8080 | 后端 REST API 服务 |
| frontend | Vue 3 + Vite + Element Plus | 3000 | 用户前台 |
| frontend-admin | Vue 3 + Vite + Element Plus + ECharts | 3001 | 管理员后台 |

```

  ├── M:N ── t_news_like (点赞新闻)

  ├── 1:N ── t_news_comment (评论新闻)

  ├── 1:N ── t_message (在线留言)

  └── 1:N ── t_survey_answer (提交问卷)

t_news (新闻)
  ├── N:1 ── t_news_category (所属分类)
  ├── M:N ── t_tag (关联标签，通过 t_news_tag)
  ├── 1:N ── t_news_comment (评论，支持嵌套回复)

              └── 1:N ── t_survey_answer_detail (答题明细)
```

## 五、功能模块详细说明

### 5.1 用户前台功能

#### 5.1.1 用户认证

**注册**
- 填写用户名、昵称、密码、确认密码
- 输入图形验证码（Redis 存储，有效期限时）
- 注册成功后跳转登录页

**登录**
- 输入用户名、密码、图形验证码
- 验证通过后返回用户信息，前端存入 localStorage
- 登录状态在页面刷新后自动恢复

#### 5.1.2 首页推荐

- 已登录用户：基于**协同过滤推荐算法**，根据用户的浏览、点赞、收藏行为推荐个性化新闻
- 未登录用户：展示最新发布的新闻列表
- 顶部展示轮播图，提供视觉引导
- 推荐新闻以时间线形式呈现，配有 GSAP 动画效果

#### 5.1.3 新闻信息

**新闻列表 (NewsList)**
- 分页浏览所有已发布新闻
- 支持多维度筛选：分类、标签、关键词、作者
- 支持多种排序：最新发布、点赞最多、收藏最多
- 新闻工作者（is_journalist=1）可发布新闻，支持富文本内容、图片上传、视频上传
- 发布的新闻状态为"待审核"，需管理员审核通过后展示

**新闻详情 (NewsDetail)**
- 展示新闻标题、作者、发布时间、浏览量、标签
- 正文以富文本 (HTML) 渲染，支持嵌入图片和视频
- 登录用户可进行：点赞/取消点赞、收藏/取消收藏、发表评论、回复评论
- 评论支持两级嵌套结构（根评论 + 子回复）
- 自动记录浏览历史

#### 5.1.4 论坛社区

- 浏览论坛帖子列表，支持标题搜索和时间范围筛选
- 登录用户可发帖（发帖后需审核）
- 登录用户可点赞/取消点赞帖子
- 帖子详情以弹窗形式查看
- 评论功能在弹窗中完成

#### 5.1.5 问卷调查

- 分页浏览问卷列表（需登录访问）
- 查看问卷详情（标题、说明、题目列表）
- 参与答题，支持题型：
  - **单选题**（question_type=1）
  - **多选题**（question_type=2）
  - **评分题**（question_type=3）
- 自动检测是否已提交，防止重复作答
- 前端校验必答题

#### 5.1.6 公告资讯

- 表格形式展示公告列表
- 点击表格行查看公告详情
- 登录用户可点赞/取消点赞公告
- 默认自动展示第一条公告

#### 5.1.7 在线留言

- 需登录才能访问
- 提交留言反馈给管理员
- 查看自己的留言列表和处理状态
- 可查看管理员的回复内容

#### 5.1.8 个人中心

- **基础信息管理**：修改昵称、上传头像
- **新闻工作者申请**：申请成为新闻工作者（管理员审核）
- **账号安全**：修改密码、申请注销账号
- **我的内容**：
  - 我的论坛帖子（查看/编辑/删除/查看评论）
  - 我的留言（查看留言及回复状态）
  - 我发布的新闻（仅新闻工作者可见，支持查看/编辑/查看审核状态）
- **浏览历史**：查看近期浏览过的新闻
- **我的点赞**：已点赞的新闻和帖子
- **我的收藏**：已收藏的新闻

### 5.2 管理员后台功能

#### 5.2.1 管理员登录

- 账号密码登录（默认：admin / admin123）
- 登录信息存入 localStorage

#### 5.2.2 数据统计仪表盘

- 展示核心数据：用户总数、新闻总数、新闻分类数、论坛帖子数
- **饼图**：各类数据占比分布
- **折线图**：互动趋势（点赞、收藏、评论），支持 7 天 / 14 天切换
- 支持导出趋势数据为 CSV 文件

#### 5.2.3 用户管理

- 分页查看用户列表，支持按 ID、用户名、昵称搜索
- 禁用/启用用户账号
- 审批新闻工作者申请（通过/拒绝）
- 处理注销申请（同意删除/拒绝恢复正常）
- 删除用户

#### 5.2.4 新闻分类管理

- 查看分类列表
- 新增/编辑/删除新闻分类
- 设置排序和状态

#### 5.2.5 新闻管理

- 分页查看新闻列表，支持按状态、分类、标签、标题筛选
- 管理员可直接发布新闻
- 支持 **JSON 批量导入**新闻
- 审核新闻：通过（status→2）/ 驳回（status→3，附驳回原因）
- 查看新闻详情（元数据 + HTML 正文）
- 编辑/删除新闻

#### 5.2.6 公告资讯管理

- 分页查看公告列表
- 新增/编辑/删除公告
- 置顶/取消置顶公告

#### 5.2.7 论坛管理

- 分页查看论坛帖子
- 审核帖子：通过/驳回（附审核意见）
- 置顶/取消置顶帖子
- 查看并管理帖子评论
- 删除违规帖子或评论

#### 5.2.8 问卷调查管理

- 分页查看问卷列表
- 新增/编辑/删除问卷
- **题目管理**：新增/编辑/删除题目（单选、多选、评分）
- **选项管理**：新增/编辑/删除题目选项
- **统计分析**：查看每道题的选项选择数量和百分比

#### 5.2.9 在线留言管理

- 分页查看所有用户留言
- 回复未处理的留言

#### 5.2.10 系统管理

- **关于我们**：编辑平台"关于我们"内容
- **友情链接**：新增/编辑/删除友情链接

## 六、API 接口设计

### 6.1 统一响应格式

所有接口统一返回 `ApiResponse<T>` 格式：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": { ... }
}
```

- `code: 0` 表示成功
- `code: -1` 或其他值表示失败

## 七、核心业务流程

### 7.1 用户注册与登录流程

```
用户 → 注册页面 → 填写信息 + 验证码
                        │
                        ▼
              后端验证 Redis 验证码
                        │
                ┌───────┴───────┐
                │ 验证通过       │ 验证失败
                ▼               ▼
          BCrypt 加密密码    返回错误
          存入 t_user       刷新验证码
                │
                ▼
          跳转登录页 → 登录 → 验证成功 → 返回用户信息
                                         │
                                         ▼
                                   存入 localStorage
                                   跳转首页
```

### 7.2 新闻发布与审核流程

```
新闻工作者 → 发布新闻 → status=1 (待审核)
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
              管理员审核通过         管理员驳回
              status=2 (已发布)    status=3 (驳回)
              设置 publish_time    附 review_remark
                    │                   │
                    ▼                   ▼
              前台可见显示        用户可查看驳回原因
                                可修改后重新提交
```

### 7.3 新闻推荐算法流程

```
用户登录 → 首页加载
              │
              ▼
    调用推荐接口 (userId)
              │
              ▼
    后端协同过滤算法处理：
    1. 收集用户行为数据（浏览/点赞/收藏）
    2. 计算用户相似度
    3. 找到相似用户喜好的新闻
    4. 排除已浏览新闻
    5. 返回推荐列表
              │
              ▼
    前端展示推荐新闻列表
    （未登录则降级为最新新闻）
```

### 7.4 问卷调查流程

```
管理员创建问卷 → 添加题目 → 添加选项 → 发布问卷
                                         │
                                         ▼
                              用户浏览问卷列表
                                         │
                                         ▼
                              查看详情 → 参与答题
                                         │
                                         ▼
                              前端校验必答题
                                         │
                                         ▼
                              提交答卷（检查重复提交）
                                         │
                                         ▼
                              管理员查看统计结果
```

## 十一、关键技术实现

### 11.1 统一响应处理

**后端** — 所有接口返回 `ApiResponse<T>`，成功 code=0，失败 code=-1。

**前端** — Axios 拦截器统一解包响应：
- 成功时直接返回 `response.data.data`（业务数据）
- 失败时 reject 错误信息

> 注意：`el-upload` 组件绕过 Axios 拦截器，需在上传回调中手动检查 `response.code === 0`。

### 11.2 密码安全

使用 Spring Security Crypto 的 **BCryptPasswordEncoder** 对用户密码进行加密存储，数据库中不保存明文密码。

### 11.3 文件上传

- 接口：`POST /api/upload/image`、`POST /api/upload/video`
- 文件保存到服务器 `uploads/` 目录
- 返回相对路径如 `/uploads/filename.ext`
- 前端开发环境通过 Vite 代理访问上传文件

### 11.4 协同过滤推荐

首页推荐算法基于用户行为数据（浏览历史、点赞、收藏），通过协同过滤计算用户相似度，推荐相似用户喜好但当前用户未阅读的新闻。

### 11.5 MyBatis-Plus 分页

使用 MyBatis-Plus 内置分页插件，返回 `Page<T>` 对象：

```json
{
  "records": [...],
  "total": 100,
  "pages": 10,
  "size": 10,
  "current": 1
}
```

### 11.6 全局异常处理

通过 `GlobalExceptionHandler`（`@RestControllerAdvice`）捕获全局异常，统一返回 `ApiResponse` 格式的错误信息，避免堆栈信息暴露。
